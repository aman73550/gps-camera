Yeh kaafi frustrating situation hoti hai jab app Expo Go mein toh smooth chalti hai, par APK banane ke baad crash hone lagti hai. Iska matlab hai ki Development Environment (Expo Go) aur Production Environment (APK) ke beech kuch configuration ka farq hai.
Yahan kuch common reasons aur unke solutions diye gaye hain, jo aap check kar sakte hain:
1. Hardcoded Localhost ya IP Address
Sabse bada kaaran ye hota hai ki aapne API calls ke liye localhost ya apne laptop ka IP address (192.168.x.x) use kiya hoga.
 * Problem: Expo Go mein phone aur laptop ek hi network par hote hain toh API kaam karti hai. APK banne ke baad localhost ka matlab phone ka apna internal address ho jata hai.
 * Solution: Humesha production API ka HTTPS URL use karein. Agar testing kar rahe hain, toh ensure karein ki server publicly accessible ho.
2. Missing Permissions in app.json
Expo Go mein saari permissions (Camera, Location, etc.) by default enabled hoti hain, lekin APK mein aapko unhe manually define karna padta hai.
 * Check: Apni app.json file dekhein aur check karein ki android.permissions mein sab kuch sahi hai ya nahi.
   "android": {
  "permissions": [
    "CAMERA",
    "ACCESS_FINE_LOCATION",
    "READ_EXTERNAL_STORAGE"
  ],
  "package": "com.yourname.appname"
}

3. API Keys (Google Maps, Firebase, etc.)
Agar aap Google Maps ya Firebase use kar rahe hain, toh unki API keys Expo Go ke liye alag hoti hain aur APK ke liye alag setup chahiye hota hai.
 * Solution: Google Cloud Console mein jaakar apni API key ko APK ke SHA-1 Fingerprint ke saath restrict karein. Bina sahi key ke, app main screen par aate hi crash ho sakti hai.
4. Environment Variables (.env)
Agar aap .env file use kar rahe hain, toh ho sakta hai ki build bante waqt wo values APK mein bundle na ho rahi hon.
 * Fix: Check karein ki aapne expo-constants use kiya hai ya nahi, aur EAS build ke waqt secrets ko configure kiya hai ya nahi.
5. Production Error (Try-Catch)
Main app mein enter hote hi crash hone ka matlab hai ki useEffect ya render function mein koi aisi cheez hai jo null return kar rahi hai.
 * Fix: Apne main component ko ek Error Boundary mein wrap karein ya har jagah try-catch lagakar dekhein ki error kahan aa raha hai.
Real Error Kaise Check Karein? (Debugging)
Jab app mobile mein crash ho rahi ho, toh aap ye command run karke dekh sakte hain ki asal mein error kya hai:
 * Apne phone ko USB se laptop se connect karein.
 * Terminal mein ye command chalaein:
   adb logcat *:E

   (Ye command aapko live error dikhayegi jo phone ke andar ho raha hai. Isse aapko exact file aur line number pata chal jayega.)
Kya aap mujhe bata sakte hain ki aapne kaun-kaun si libraries (jaise Maps, Firebase, ya Auth) use ki hain? Isse main aapko aur specific solution de paunga.
